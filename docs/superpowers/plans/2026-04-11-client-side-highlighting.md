# Client-Side Syntax Highlighting Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace goldmark-highlighting/chroma server-side syntax highlighting with highlight.js running in the browser, using Vite as the frontend build tool.

**Architecture:** Go renderer emits plain `<pre><code class="language-xxx">` markup via goldmark's default fenced-code renderer. The browser imports highlight.js as an ES module and highlights each code block after rendering and after SSE-triggered re-renders. Vite owns the frontend build (Tailwind via PostCSS + JS bundling), emitting fixed-name files into `web/static/` which are embedded by Go and committed to the repo so `go install` keeps working.

**Tech Stack:** Go, goldmark, Vite 5, PostCSS, Tailwind CSS v3, highlight.js (common bundle), @tailwindcss/typography.

**Spec:** `docs/superpowers/specs/2026-04-11-client-side-highlighting-design.md`

	}
